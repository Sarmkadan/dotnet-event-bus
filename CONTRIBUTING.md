# Contributing to DotnetEventBus

Contributions are welcome! Please follow these guidelines:

1. **Fork and Clone**: Fork the repository on GitHub and clone your fork.
2. **Create a Branch**: Use a descriptive name (e.g., `feature/amazing-feature`).
3. **Make Changes**:
   - Keep commits small, descriptive, and follow conventional commits.
   - Add/update unit tests for any new or modified functionality.
   - Include XML doc comments for all new public APIs.
4. **Run Checks**: Ensure `dotnet build` and `dotnet test` pass.
5. **Submit a Pull Request**: Open a PR against `main` and link relevant issues.

## Style & Standards
- Follow the existing `.editorconfig` rules.
- Keep C# methods focused; use async/await for I/O operations.

## Reporting Issues
Use GitHub Issues to report bugs or request features. Include a minimal reproduction case and your .NET SDK version.
